Use Azure metric evidence for Container App resource `/subscriptions/sub-devpolaris-training/resourceGroups/rg-devpolaris-app-prod/providers/Microsoft.App/containerApps/ca-devpolaris-orders-prod`. Focus on request volume, failed requests, request duration, and replica count for `devpolaris-orders-api`.

Your job:

1. **Inspect** request and failure metrics.
2. **Inspect** latency and replica metrics.
3. **Keep** the evidence tied to the named runtime resource.

The grader checks that you collected the small set of operating signals for the API.
