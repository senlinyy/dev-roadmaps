The orders service needs a production resource group for resources that share the same lifecycle. Create `rg-devpolaris-orders-prod` in `eastus` with tags `team=orders`, `env=prod`, and `service=orders-api`.

Your job:

1. **Create the resource group** named `rg-devpolaris-orders-prod`.
2. **Place it in** `eastus`.
3. **Apply the tags** `team=orders`, `env=prod`, and `service=orders-api`.

The grader checks the Azure resource group state, not a prose explanation.
