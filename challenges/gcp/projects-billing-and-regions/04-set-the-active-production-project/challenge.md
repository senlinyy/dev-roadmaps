Set the active Google Cloud project before the Orders team runs production setup commands. The target project is `dp-orders-prod`, and the project list already includes that project.

Your job:

1. **Set the active project** to `dp-orders-prod`.
2. **Verify the selected project** after changing the CLI context.
3. **Keep the work inside** the production project boundary.

The grader checks simulated Google Cloud project state, not a prose explanation.
