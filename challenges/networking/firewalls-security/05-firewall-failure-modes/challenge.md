Firewall outages usually come from rule order, state tracking, cloud firewall directionality, persistence gaps, or a chain that does not match the packet path. Use the provided rule snippets and symptoms to identify the failure mode before recommending a change. Treat each question as a production review where the safest answer is the one the evidence supports.
