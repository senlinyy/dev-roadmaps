---
title: "Pinpoint Which Layer Broke from Tool Output"
sectionSlug: where-each-layer-breaks
order: 4
---

Real outages leave fingerprints in command output: `NO-CARRIER`, failed neighbor entries, missing routes, refused ports, TLS errors, and HTTP status codes. Read each snippet as incident evidence and decide which layer failed first. Focus on what the output proves, not on the broad phrase "the network is down."
