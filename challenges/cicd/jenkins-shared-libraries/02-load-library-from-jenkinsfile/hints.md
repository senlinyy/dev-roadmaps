The `@Library('name@ref') _` annotation goes at the top of the Jenkinsfile. The trailing `_` is required Groovy syntax and stands for "import as side-effect". Once the library is loaded, you can call any global step from `vars/` directly. Pass arguments as a Groovy named-argument map: `buildJavaService(service: 'orders', mavenGoals: [...], agentLabel: '...')`. The whole inline `pipeline { ... }` block goes away because the library renders it.
