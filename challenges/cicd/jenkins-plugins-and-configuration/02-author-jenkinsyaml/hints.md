CasC YAML mirrors Jenkins's internal config structure. `jenkins:` is the top-level key; nested mappings configure each subsystem. `users:` under `securityRealm.local:` is a list of `{ id, password }` objects. `authorizationStrategy.roleBased.roles.global` is a list of objects with `name`, `pattern` (optional), `permissions` (list), and `assignments` (list of user ids). The grader does not care about indentation as long as the YAML parses.
